# VC-3 ↔ STM-1 FPGA 实现 7个关键处理点

> **文档主题**: VC-3 (34M) 与 STM-1 (155M) 映射/解映射的 FPGA 关键实现点
> **协议依据**:
> - ITU-T G.707 §10.1.2.2 (E3异步映射到C-3)
> - ITU-T G.707 §8.2 (TU-3指针)
> - ITU-T G.707 §9.2/9.3 (SOH/POH开销)
> - ITU-T G.783 §12 (低阶通道功能块)
> **创建日期**: 2026-04-07

---

## 关键处理点总览

VC-3 (34M) ↔ STM-1 (155M) 的 FPGA 实现涉及 **7个关键处理点**：

| 序号 | 处理点 | 核心功能 | 难点等级 |
|:----:|:-------|:---------|:--------:|
| 1 | **C-3 码速调整** | E3(34M) ↔ C-3(48M) 速率适配 | ⭐⭐ |
| 2 | **VC-3 POH 编解码** | J1/B3/C2/G1 等开销处理 | ⭐⭐⭐ |
| 3 | **TU-3 指针处理** | H1/H2/H3 动态指针调整 | ⭐⭐⭐⭐⭐ |
| 4 | **TUG-3 列交错复用** | 3×TU-3 → VC-4 列间插 | ⭐⭐⭐ |
| 5 | **VC-4 到 AU-4 指针映射** | AU-4 指针解释/生成 | ⭐⭐⭐⭐ |
| 6 | **BIP 误码校验链** | B1→B2→B3 级联校验 | ⭐⭐⭐⭐ |
| 7 | **时钟域跨域 FIFO** | 多时钟域数据缓冲 | ⭐⭐⭐ |

## 1. C-3 码速调整 (Rate Adaptation)

### 1.1 基本参数

| 参数 | 值 | 说明 |
|------|-----|------|
| E3 输入速率 | 34.368 Mbit/s ± 20ppm | PDH 标称速率 |
| C-3 帧速率 | 48.384 Mbit/s | 9行×84列×8bit/125μs |
| 调整方式 | 正/负/零调整 | C2字节控制 |
| 调整机会 | 5组 (5×6字节) | 每组含调整控制比特 |
| 填充比例 | ~29% | (48.384-34.368)/48.384 |

### 1.2 C-3 帧结构 (G.707 §10.1.2.2, Figure 10-5)

**参考来源**: ITU-T G.707 §10.1.2.2 "Asynchronous mapping of 34 368 kbit/s"

```
C-3 帧: 9行 × 84列 = 756字节 (125μs周期)
═══════════════════════════════════════════════════════════════════

VC-3 帧结构 (9子帧, 每125μs):
┌─────────────────────────────────────────────────────────────────┐
│  每子帧 (84字节 = 672 bits) 包含:                               │
│  • VC-3 POH:  1 byte  (第1列)                                   │
│  • 数据位:    1431 bits (D)                                     │
│  • 码速调整控制位: 2组×5 bits (C1, C2)                          │
│  • 码速调整机会位: 2 bits (S1, S2)                              │
│  • 固定填充位: 573 bits (R)                                     │
└─────────────────────────────────────────────────────────────────┘

码速调整控制机制:
• C1C1C1C1C1 = 00000 → S1 是数据位 (承载E3数据)
• C1C1C1C1C1 = 11111 → S1 是码速调整位 (填充位, 无数据)
• C2 同理控制 S2

接收端使用 5-bit 多数表决解码 C bits，抗单/双比特误码。

注: 不是用 C2 字节做控制，而是分散在 C-3 帧中的 justification control bits!
```

### 1.3 FPGA 实现要点

| 实现项 | 规格 |
|--------|------|
| 异步 FIFO | 深度16-32字节，宽度8bit |
| 写时钟 | 34.368 MHz (E3 恢复时钟) |
| 读时钟 | 48.384 MHz (C-3 系统时钟) |
| C bits 生成 | 每帧计算调整需求，生成 C1/C2 控制码 (5-bit组) |

**关键纠正**:
- **C2 字节** = VC-3 POH 信号标签字节 (固定值 0x1a)
- **C bits** = 码速调整控制比特 (分散在 C-3 帧各处，非独立字节)

## 2. VC-3 POH 编解码

### 2.1 POH 字节功能

| 字节 | 位置 | 功能 | FPGA 处理复杂度 |
|------|------|------|-----------------|
| **J1** | 第1行 | 通道踪迹 (16字节复帧) | 高 - 需RAM+CRC7 |
| **B3** | 第2行 | BIP-8 误码监测 | 中 - 8位异或链 |
| **C2** | 第3行 | 信号标签 (0x1a=34M) | 低 - 常量寄存器 |
| **G1** | 第4行 | REI (b1-4) + RDI (b5) | 中 - 误码计数器 |
| **F2** | 第5行 | 用户通道 | 低 |
| **H4** | 第6行 | 复帧指示 (500μs) | 低 - 模4计数器 |
| **F3** | 第7行 | 用户通道 | 低 |
| **K3** | 第8行 | 低阶通道 APS | 中 - 保护协议 |
| **N1** | 第9行 | TCM/运营者字节 | 中 - 可选实现 |

### 2.2 J1 复帧结构 (关键)

```
J1 复帧: 16帧 × 4字节 = 64字节
═══════════════════════════════════════════════════════════════════

帧0:  [CRC7] [0] [API0] [API1]   - CRC7覆盖前一复帧
帧1:  [0]    [0] [API2] [API3]
帧2:  [0]    [0] [API4] [API5]
...
帧15: [0]    [0] [API28] [API29] - 30字符API标识

CRC7 生成多项式: G(x) = x^7 + x^3 + 1
```

### 2.3 B3 BIP-8 计算

- **计算范围**: 整个 VC-3 前一帧 (9行×85列=765字节)
- **插入位置**: 当前帧第2行第1列
- **时序要求**: 125μs内完成765字节异或计算
- **FPGA实现**: 8位并行异或树，单时钟周期处理1字节

## 3. TU-3 指针处理 (⭐⭐⭐⭐⭐ 核心难点)

### 3.1 指针字结构 (H1/H2)

```
H1 + H2 组成16位指针字:
═══════════════════════════════════════════════════════════════════

 b1  b2  b3  b4  b5  b6  b7  b8 | b9  b10 b11 b12 b13 b14 b15 b16
 N   N   N   N   S   S   I   D  | I   D   I   D   I   D   I   D
 └───┴───┴───┘   └─┘   └─┘ └─┘   └─┘ └─┘ └─┘ └─┘ └─┘ └─┘ └─┘ └─┘
    NDF(4bit)   SS  I-bit   D-bit   I   D   I   D   I   D   I   D

 N-bits (b1-b4): 新数据标志 (0110=正常, 1001=NDF)
 SS-bits (b5-b6): AU类型 (已废弃)
 I-bits (b7,b9,b11,b13,b15): 正调整指示位 (5位多数表决)
 D-bits (b8,b10,b12,b14,b16): 负调整指示位 (5位多数表决)

 指针值 = b7-b16 的10位二进制值 (TU-3: 0-764, AU-4: 0-782)
```

### 3.2 指针调整规则

| 操作 | I-bits 状态 | D-bits 状态 | 动作 |
|------|-------------|-------------|------|
| **正调整** | ≥3个为1 | 无要求 | H3后为填充字节，指针+1 |
| **负调整** | 无要求 | ≥3个为1 | H3携带VC-3数据，指针-1 |
| **NDF** | =0110 | =0110 | 正常操作 |
| **新数据** | =1001 | =1001 | 忽略I/D，直接加载新指针值 |

### 3.3 关键约束

| 参数 | 值 | 说明 |
|------|-----|------|
| 调整间隔 | ≥3帧 | 连续调整必须间隔至少3帧 |
| 指针解释 | 3次一致 | 新值需连续3帧相同才确认 |
| H3位置 | 第4行第85-87列 | VC-4帧中的绝对位置 |
| FIFO深度 | 8-16字节 | 吸收指针调整引起的抖动 |

### 3.4 状态机设计

```
┌──────────┐    时钟偏差>阈值    ┌──────────┐
│  NORMAL  │─────────────────────→│  INC_J   │
│  (正常)  │←─────────────────────│ (正调整) │
└────┬─────┘    I-bits多数=1      └────┬─────┘
     │                                 │
     │ 时钟偏差<-阈值                  │ 写入填充字节
     ▼                                 ▼
┌──────────┐    D-bits多数=1    ┌──────────┐
│  DEC_J   │←───────────────────│   执行   │
│ (负调整) │────────────────────→│ 调整操作 │
└────┬─────┘    写入H3数据       └────┬─────┘
     │                                 │
     └─────────────────────────────────┘
              返回 NORMAL (延迟3帧)
```

## 4. TUG-3 列交错复用

### 4.1 映射关系

```
3个 TU-3 → 1个 VC-4 (列间插)
═══════════════════════════════════════════════════════════════════

VC-4: 9行 × 261列

TU-3 #1 (85列) → VC-4 列 4,7,10,13...259 (间隔3列)
TU-3 #2 (85列) → VC-4 列 5,8,11,14...260 (间隔3列)
TU-3 #3 (85列) → VC-4 列 6,9,12,15...261 (间隔3列)

公式: VC-4_col = 4 + TU_num + 3 × (TU_col-1)
      TU_col = 1..86, TU_num = 0,1,2
```

### 4.2 FPGA 实现

| 模块 | 功能 | 资源 |
|------|------|------|
| 3路 TU-3 缓冲 | 每帧765字节存储 | 3×1KB RAM |
| 列交错器 | 地址映射: col_out = col_in×3 + offset | LUT逻辑 |
| VC-4 RAM | 261列×9行缓冲 | 2-3KB RAM |

## 5. VC-4 到 AU-4 指针映射

### 5.1 AU-4 与 TU-3 指针对比

| 参数 | TU-3 指针 | AU-4 指针 |
|------|-----------|-----------|
| 位置 | VC-4 第4行 (TU-3级) | STM-1 第4行 SOH |
| 值范围 | 0-764 (10bit) | 0-782 (10bit) |
| 调整单位 | 1字节 | 3字节 (VC-4级) |
| H3 作用 | 1字节负调整 | 3字节负调整 |
| 级联关系 | 受AU-4指针变化影响 | 独立控制 |

### 5.2 级联指针处理

```
AU-4 指针变化 → 级联影响 TU-3 指针
─────────────────────────────────────────

当 AU-4 做正/负调整时:
- VC-4 在 AU-4 中的位置改变
- 但 TU-3 在 VC-4 内的相对位置不变
- 因此 TU-3 指针值**不需要**因AU-4调整而改变

例外: AU-4 NDF (新数据标志) 导致指针跳变
      → 需重新解释 TU-3 指针
```

## 6. BIP 误码校验链

### 6.1 三级 BIP 级联

```
SDH BIP 校验链 (发送方向):
═══════════════════════════════════════════════════════════════════

B3 (VC-3 POH) → B2 (MSOH) → B1 (RSOH)
     ↑              ↑            ↑
   765字节      2349字节     2430字节
   VC-3范围     VC-4范围     STM-1范围

接收方向: 反向验证
```

### 6.2 各级 BIP 参数

| BIP | 计算范围 | 宽度 | 插入时机 | FPGA资源 |
|-----|----------|------|----------|----------|
| **B3** | VC-3 (9×85=765B) | BIP-8 | 扰码前 | 8位LFSR |
| **B2** | MSOH+VC-4 (9×261=2349B) | BIP-24 | 扰码前 | 24位并行异或 |
| **B1** | STM-1全帧 (9×270=2430B) | BIP-8 | 扰码前 | 8位LFSR |

### 6.3 关键时序

- **计算延迟**: 需完整接收前一帧后才能计算
- **存储需求**: 需1帧缓冲保存计算结果
- **插入位置**: 当前帧对应开销字节位置

## 7. 时钟域跨域 FIFO

### 7.1 时钟域分布

```
VC-3 ↔ STM-1 时钟域分布
═══════════════════════════════════════════════════════════════════

E3 RX (34.368M) ─────┐
                     ├──→ [FIFO 1] ──→ C-3 (48.384M)
E3 TX (34.368M) ─────┘        │
                              ▼
                         VC-3处理
                              │
                         [FIFO 2] ──→ AU-4 (155.520M)
                              ▼
                         STM-1线路
```

### 7.2 FIFO 配置

| 位置 | 深度 | 宽度 | 用途 | 时钟域 |
|------|------|------|------|--------|
| E3↔C-3 | 16-32字节 | 8bit | 码速调整 | 34M↔48M |
| 指针调整 | 8-16字节 | 8bit | 吸收指针抖动 | 48M↔155M |
| TX总线 | 2-4帧 | 8bit | 帧对齐 | 异步 |

### 7.3 跨时钟域设计

| 技术 | 应用场景 |
|------|----------|
| 异步FIFO (双端口RAM) | 大数据量缓冲 |
| 握手同步器 | 控制信号跨域 |
| 脉冲同步器 | 单比特事件 |
| Gray码计数器 | FIFO读/写指针同步 |

### C-3 码速调整参考来源详解

**主要参考**: ITU-T G.707/Y.1322 (01/2007) §10.1.2.2 "Asynchronous mapping of 34 368 kbit/s"

> "One 34 368 kbit/s signal can be mapped into a VC-3 as shown in Figure 10-5.
> In addition to the VC-3 POH, the VC-3 consists of a payload of 9 × 84 bytes every 125 µs.
> This payload is divided in three subframes, each subframe consisting of:
> – 1431 data bits (D);
> – two sets of five justification control bits (C1, C2);
> – two justification opportunity bits (S1, S2);
> – 573 fixed stuff bits (R)."

**关键参数确认**:
- C-3 帧周期: 125 μs (同 VC-3)
- C-3 帧大小: 9 × 84 = 756 字节
- 标称速率: 48.384 Mbit/s
- 码速调整: 正/负/零调整通过 C1/C2 bits 控制 S1/S2 bits
